<product_requirements>
The goal is to create a comprehensive, public-facing website to browse and analyze a historical database of Yu-Gi-Oh! Shonen Jump Championship (SJC) decklists from 2004-2010. The application should be fast, visually appealing, and provide powerful tools for data exploration, closely mirroring a reference Notion database provided by the user.

**Implemented Features:**
-   **Homepage:** A modern, asymmetrical bento grid layout providing access to all major site sections.
-   **Decklists Database:** A paginated view of all tournament decklists.
-   **Card Browser:** A detailed, Notion-style table of individual card usage records across all decks. It features multi-level sorting (Year, Deck Name, etc.), real-time search, and filtering.
-   **Decklist Viewer:** A dedicated page and a quick-preview side panel to view full decklists, with cards grouped by type (Monster, Spell, Trap).
-   **Master Lists Hub:** A central page linking to various master lists for collection tracking.
-   **Interactive Checklists:** For the master lists, users can check off cards they own. This progress is saved locally in their browser's  and visualized with a quantity-based completion percentage.
-   **UI/UX:** A custom Modern Championship color scheme (fuchsia/gold/dark navy) and a polished interface with features like a Reset Progress confirmation and a scroll-to-top button.
</product_requirements>
<key_technical_concepts>
- **Full-Stack Application:**
    - **Backend:** FastAPI (Python)
    - **Frontend:** React (JavaScript)
    - **Database:** MongoDB
- **Data Ingestion:** Python scripts parsing user-provided CSV files to populate MongoDB collections.
- **REST API:** Endpoints for fetching decklists, card usage data, and master lists with support for pagination, filtering, and complex multi-level sorting.
- **Client-Side Persistence:** Using browser  to save user progress on checklists without requiring user accounts.
- **UI Framework:** TailwindCSS for styling and layout.
</key_technical_concepts>
